	3. **traceroute -n**:
	   Ця утиліта покаже шлях пакету по всіх вузлах мережі від відправника до точки призначення. Прапорець ```-n``` покаже адреси тільки у номерному форматі (тобто у форматі звичайних ip адрес). Запит можна робити як і на доменне імʼя, так і на конкретну адресу. Якщо вказати імʼя, а воно буде привʼязано до декількох ip адрес (як видно з результату виконання команди dig), то утиліта автоматично вибере одну з привʼязних. Або ж можна зробити запит напряму на ip адресу:![](resources/pictures/Знімок%20екрана%202026-07-19%20о%2013.52.51.png)
	   Як видно, пакету довелось пройти 16 хопів для кінцевої точки призначення. Хопи з 11 по 16 не відображені бо, скоріше за все, це внутрішні вузли мережі Google, які не повертають інформацію про прийнятий пакет, тому інформація про ці стрибки не повертаєтсья і не відображається. 

2. **Зніміть tcpdump під час curl. Знайдіть у виводі SYN, SYN-ACK, ACK**
   Під час запуску ```tcpdump``` без параметрів, утиліта починає перехоплювати увесь трафік, через що логи швидко стають нечитаємими. Для спрощення читання треба обрати конкретиний інтерфейс, з якого будуть іти дані. Так як я буду робити запит на **API**, яке лежить на моєму віддаленому сервері, то я це буду робити по WIFI, отже, треба слухати лише інтерфейс WIFI модему. Для цього можна використати інформацію, отримату з результату виконання команди ```route get``` з пункту ***1.2***
   Бездротовий інтерфейс має назву ```en0```, отже, запустимо прослуховування саме цього інтерфейсу. Логів все одно багато, тому спростимо задачу ще більше. Конкретизуємо хост, на який будемо відправляти запит.

3. **Запишіть у файл: скільки хопів, який ваш шлюз, який MAC у шлюзу**

   ```bash
	   #!/bin/bash
	GATEWAY=$(route -n get default | grep gateway | awk '{print $2}')
	echo "Gateway: $GATEWAY" > network_data.txt
	echo "MAC-addres of gateway: $(arp -n $GATEWAY | awk '{print $4}')" >> network_data.txt
	echo "=== Hops to server ===" >> network_data.txt
	traceroute allert-depending-alarm.pp.ua >> network_data.txt
	cat network_data.txt
   ```

2. **Поясніть письмово в 5 реченнях, чому MAC змінюється, а IP — ні**
   Тому що IP двох вузлів — це їх ідентифікатори в мережах, а MAC-адреса — це фізична адреса пристрою. Під час доставки пакета з одного пристрою до іншого, вони проходять через різну кількість мережевих вузлів — це певні пристрої в даній мережі. Машина клієнта, шлюз, шлюз провайдера (зі сторони клієнта і сервера), шлюз локальної мережі сервера та сам сервер. І кожен з цих пристроїв має свою MAC адресу. Тому на кожному хопі MAC адреси міняються, а IP — ні, тому що IP - це ідентифікатор кінцевої точки в мережі, а MAC - ідентифікатори лише двох сусідів у цій же мережі
